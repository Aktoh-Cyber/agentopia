# Generated Agents

This directory contains agents automatically generated by the Agent Generator system.

## Structure

```
generated-agents/
├── router-agents/           # Router agents that route to specialists
│   └── [agent-name]/
│       ├── src/entry.py
│       ├── wrangler.toml
│       └── .agent-metadata.json
└── specialist-agents/      # Specialist agents for specific domains
    └── [agent-name]/
        ├── src/entry.py
        ├── wrangler.toml
        └── .agent-metadata.json
```

## Automated Workflow

1. **Generation**: Agent Generator creates new agents in feature branches
2. **Validation**: GitHub Actions validate configuration and test builds
3. **Deployment**: Successful validation triggers Cloudflare Workers deployment
4. **Cleanup**: Removed agents are automatically cleaned up from Cloudflare

## Metadata Format

Each agent includes `.agent-metadata.json` for tracking:
```json
{
  "generatedAt": "2025-01-11T12:00:00Z",
  "workerName": "agent-name",
  "domain": "agent.domain.com",
  "type": "router|specialist",
  "language": "python|javascript",
  "accountId": "cloudflare-account-id",
  "zoneId": "cloudflare-zone-id",
  "dependencies": ["dependent-agent-names"]
}
```

## Manual Operations

⚠️ **Do not manually edit files in this directory**. All changes should be made through the Agent Generator to maintain consistency and proper CI/CD workflows.

For agent modifications:
1. Use the Agent Generator API to create updated configuration
2. The system will create a new PR with changes
3. Review and merge through normal GitHub workflow
